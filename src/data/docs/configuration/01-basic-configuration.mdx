Learn how to create a basic Haloy configuration file to deploy your application.

## Minimal Configuration

The simplest `haloy.yaml` configuration:

<Code code={`
  name: "my-app"
  server: haloy.yourserver.com
  domains:
    - domain: "my-app.com"
      aliases:
        - "www.my-app.com"
`} lang="yaml" />

This configuration:

- Looks for a Dockerfile in the same directory
- Builds the image locally
- Uploads it to your server
- Deploys with SSL/TLS enabled
- Redirects www to the main domain

## Format Support

Haloy supports YAML, JSON, and TOML for configuration files.

### YAML (`*.yml` or `*.yaml`)
<Code code={`
  name: "my-app"
  server: haloy.yourserver.com
  domains:
    - domain: "my-app.com"
      aliases:
        - "www.my-app.com"
  health_check_path: "/health"
`} lang="yaml" />

### JSON (`*.json`)
<Code code={`
  {
    "name": "my-app",
    "server": "haloy.yourserver.com",
    "domains": [
      {
        "domain": "my-app.com",
        "aliases": ["www.my-app.com"]
      }
    ],
    "health_check_path": "/health"
  }
`} lang="json" />

### TOML (`*.toml`)

<Code code={`
  name = "my-app"
  server = "haloy.yourserver.com"
  health_check_path = "/health"

  [[domains]]
  domain = "my-app.com"
  aliases = ["www.my-app.com"]
`} lang="toml" />

**Naming Convention:**

- YAML/TOML: Use `snake_case` (e.g., `health_check_path`)
- JSON: Use `camelCase` (e.g., `healthCheckPath`)

## Adding a Domain

To make your app accessible via a domain:

<Code code={`
  name: "my-app"
  server: haloy.yourserver.com
  domains:
    - domain: "my-app.com"
`} lang="yaml" />

## Multiple Domains and Aliases

<Code code={`
  domains:
    - domain: "my-app.com"
      aliases:
        - "www.my-app.com"
        - "app.my-app.com"
    - domain: "another-domain.com"
`}
  lang="yaml"
/>

## Custom Port

If your application listens on a port other than 8080:

<Code code={`
  name: "my-app"
  server: haloy.yourserver.com
  port: "3000"  # Your app's internal port
`}
  lang="yaml"
/>

The proxy will route traffic from ports 80/443 to this container port.

## Health Checks

Configure a custom health check endpoint:

<Code code={`
  name: "my-app"
  server: haloy.yourserver.com
  health_check_path: "/api/health"
`} lang="yaml" />

Default: `"/"`

## Configuration File Location

By default, `haloy` looks for configuration in the current directory:

- `haloy.yaml`
- `haloy.yml`
- `haloy.json`
- `haloy.toml`

Specify a custom config file:

<Code code={`haloy deploy --config /path/to/config.yaml`} lang="bash" />
